- [x] **1.7** Property-name blacklist added to the transform interpreter's `member` evaluator at [transformRuntime.ts](entrypoints/background/transformRuntime.ts). Threat model clarification first — the engine is NOT `eval`-based; it's a handwritten tokenizer + tree-walking interpreter. The escape vector is that native JS `String` / `Number` / `Boolean` are exposed in scope (for legitimate use: `String(x)` to cast), and the `member` evaluator walks arbitrary property chains unchanged — so `String.constructor("...")()` reaches the real `Function` constructor and compiles arbitrary code inside the sandbox worker. Blast radius capped by worker isolation (no `chrome` API, no page DOM) but still permits fetch-based exfiltration of transform inputs AND forged transform output. Batch 5's taint-redaction already replaces tainted `vars` with `[REDACTED]` before entry, so secrets don't cross the boundary, but non-secret sensitive fields (row contents, input text) remain in the worker. **Fix (Option A only — user-approved scope):** 9-name `FORBIDDEN_PROPERTY_NAMES` set blocked at the `member` evaluator right after the property key is normalized to a string — `constructor`, `prototype`, `__proto__`, `__defineGetter__`, `__defineSetter__`, `__lookupGetter__`, `__lookupSetter__`, `caller`, `arguments`. Normalizing to string first (via `String(property)`) means dot-notation, computed access (`["constructor"]`), concat-obfuscated names (`["cons" + "tructor"]`), and literal escapes all land in the same check. **Verified via standalone smoke script** against 6 escape PoCs (dot / bracket / `.prototype.constructor` chain / concat / `helpers.trim.constructor` indirect / `__proto__`) — all throw `Forbidden property access: <name>` — and 6 legit cases (`String(row.x)`, `helpers.upper(input)`, `row.name` dot + bracket, chained helpers) — all return expected values. Member ASSIGNMENT is not a parser production (no `row.__proto__.x = y` path) so prototype pollution is already impossible; the `__proto__` ban is defense-in-depth for any future assignment-capable parser. **Explicitly out of scope:** identifier whitelist (user-approved Option A-only; the known PoC is fully closed by property-name blacklist alone and an identifier whitelist would add tangential coverage at cost of broader diff); removing native `String`/`Number`/`Boolean` from scope (would require a back-compat shim + helper redirect, deferred); QuickJS-wasm sandbox swap (~300-500KB bundle + CSP `wasm-unsafe-eval` implications, deferred). compile ✓.
